Evenly distributes pattern data among multiple threads for hd processing.

This algorithm looks for available processing threads and if there are enough patterns to distribute and available threads then new threads are recursively dispatched and the current thread just listens and waits for the dispatched threads to finish processing.

Parameters
newPatternCountnumber of patterns found in the current level search
morePatternsToFindboolean indicating patterns have been found or not
fileListhark disk processing data
levelInfocurrent level processing information
isThreadDefunctedindicates if the thread is deactivated and there has been recursively spawned threads to improve processing
Returns
bool indicating the threads have been dispatched or not

Algorithm that computes whether the next level uses hd or ram.

Factors in the previous level's mode of processing and the current level's intended mode of process and computes the resources needed to process within the computer's limited ram

Parameters

sizeOfStringsize of the patterns for the level
Returns
bool indicating whether ram or hd is used, true is hd, false is ram

Keeps track of the state of each processing thread.

Monitors threads and releases processing holds on a thread when a thread has finished processing. This allows other threads that still have work to do to utilize an extra thread for their processing. It is a simplified implementation of a thread pool.

Parameters
localWorkingThreadsindexes into the localWorkingThreads that need to be monitored
localThreadPoolvector containing threads to monitor
recursiveindicating if these threads were dispatched recursively or not
threadcurrent thread being used
